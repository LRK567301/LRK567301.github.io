윈도우 Photofield에 FFmpeg랑 exiftool 등록

3월 17, 2025

https://github.com/SmilyOrg/photofield

configuration.yaml을 아무리 바꿔봐도 사진 인덱싱하다 터지길래

Unable to generate image thumbnail all generators failed: ffmpeg binary not found extension not supported : E:\Documents\Pictures\Mobile\Camera_2018_XZ1\MOV_0846.mp4
Unable to decode image thumbnail unable to decode unexpected EOF
Unable to load image info meta image: unknown format E:\Documents\Pictures\Mobile\Screenshots_2018_S9\ガルパ\ガルパ_2019-11-14-08-51-14.mp4
Unable to load image info meta tiff: unsupported feature: color model E:\Documents\Pictures\Mobile\Camera_2013_N5\IMG_20150423_025849_1.dng

쏟아지는 파일 에러들 탓인가 싶어 FFmpeg랑 exiftool를 추가했다. 이거부터 한참 헤맸어. PATH가 윈도우 환경변수 PATH인걸 이해를 못하고 다른 윈도우 유틸처럼 ffmpeg.exe를 같은 폴더에 넣어주고 있었다. 그런 멍청한 짓 그만두고 ffmpeg 적당한 폴더에 두고 환경변수에 경로 추가, exiftool는 공홈 설명대로 CLI에서 쓰려면 exiftool(-k).exe를 exiftool.exe로 바꿔놓고 똑같이 경로 추가. 윈10까지는 explorer.exe만 재시작해도 환경변수 인식이 됐던 모양인데 윈11에선 안되더라. 재부팅.

하지만 이번에도 index metadata는 넘어갔는데 중간중간 Unable to decode image thumbnail unable to decode unexpected EOF 몇개 띄우다

index contents   29% completed, 65612 loaded, 154950 pending, 53.93 / sec, 48m43s left
panic: runtime error: slice bounds out of range [:6987] with capacity 512

쯤에서 패닉해버렸다. index metadata도 CJK까지 어지간히 넘어가는데 (´･ω･`) 같은 특수문자들이 Unable to load image info meta Filename contains control characters를 몇몇 띄웠더라. exiftool만 놓고보면 유니코드 어쩌구나 사진 파일명 자체를 바꿔서 대응도 가능하겠지만 그런게 없어도 터지진 않아야 할 시점에서 터지니까 무리.

내 환경에서 immich를 가장 대체해줄만한 툴인데 아쉽다. 괜히 Experimental fast photo viewer가 아니야.

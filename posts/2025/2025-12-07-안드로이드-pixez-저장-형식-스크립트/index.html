[안드로이드] PixEz Flutter 저장 형식 스크립트

오래 잘 써온 써드파티 픽시브 클라이언트 Pixiv-Shaft의 UI가 갑자기 완전히 바뀌더니 갖다박은 가시성, 갖다박은 사용성으로 돌아왔다. 리퀴드 글래스에서 잘못된 영감을 받은 것 같은 투명 UI 뒤에 무작위 일러스트가 배경으로 깔리는 건 그렇다 치고 팔로우 신착 목록 왜 이따위야? 격자식 목록이 아니라 유저 단위 묶음이라니 이건 팔로우 몇명 안될때면 몰라 바꿀 방법도 없어.

당연히 나만 이렇게 느낄 리 없으니 반발이 컸고 잠깐 업데이트되다가 유기되더니 다행히 돌아는 왔다. 근데 v4.4.0-classic 분리에 생계 팔이까지? 오래 고마웠지만 미래가 안 보이는 앱이 되어버렸어.

그래서 깃헙 픽시브 랭킹 바로 위인 PixEz Flutter으로 갈아탔다. 한국어 메뉴는커녕 영문 지원도 시원찮은 Shaft 대비 여러모로 괜찮은 부분들이 많은데 순정 설정으로는 유저숫자ID_유저명 폴더를 만들 수가 없어서 제대로 써보진 않았거든.

https://github.com/Notsfsssf/pixez-flutter/issues/1006

근데 스크립트를 쓰면 되는구나? 기본 옵션으로 되면 편하지만 스크립트로 더 강력하게 커스텀되면 오히려 좋지.

function sanitize(name) {
  return name
    // 윈도우에서 금지되는 문자들 → 언더스코어
    .replace(/[\/\\:*?"<>|]/g, "_")
    // 끝의 점(.)이나 공백 제거
    .replace(/[. ]+$/g, "");
}

function eval(illust, index, mime) {
  const userId = illust.user.id;
  const userName = sanitize(illust.user.name);
  const illustId = illust.id;
  const title = sanitize(illust.title);
  const part = "p" + index;

  return `${userId}_${userName}/${userId}_${illustId}_${title}_${part}.${mime}`;
}

시행착오(파일명만 처리해서 작가명이 하위폴더로 나뉜다거나)를 거쳐 기존 문제점까지 보완했다. Pixiv-Shaft도 작가명에 슬래시 들어간 것 정도는 처리해줬는데 뒤에 점 붙은건 그대로 넘겼거든. 그걸 폴더싱크와 윈도우 SMB가 그대로 받아들이면서 에브리씽으로 ‘숫자_작가명’이랑 ‘숫자_작가명.’ 둘이 동시에 검색은 되는데 둘 다 열면 ‘숫자_작가명’으로만 연결됐다. 파일이 넘어는 왔지만 7zip 탐색기나 wsl을 거쳐야 했어. 그걸 파일명은 물론 폴더명까지 시작부터 개선. %2로 대체됐던 슬래시가 깔끔하게 언더바로 통일됐다.

전체적인 사용성도 뛰어나다. 한국어 메뉴 완벽하고, 썸네일 아래 제목 표시도 가독성 좋고, 다운로드 스텝은 한 단계 깊지만 이미지 선별은 훨씬 빠르고, 최근에야 활용하기 시작한 북마크=다운로드 옵션이면 결국 다를 것도 없다. 큰 틀, 큰 기능은 그대로야. 약간 다른건 우고이라 기본처리. 북마크만 눌러서는 정지이미지 한장만 받고 다른 옵션은 gif 인코딩/zip 내보내기를 따로 선택해야 한다. 기능성은 뛰어난데 작가명 폴더는 커녕 작품ID로만 저장되는건 아쉬운 부분.

하지만 스크립트 붙잡고 코파일럿이랑 몇번 상담한 것만으로 훨씬 만족스럽다. 윈도우 smb를 통해 이미 저장된 작가명 슬래시가 하위폴더로 나뉘거나 끝의 점이 붙은 폴더도 파워쉘 스크립트로 금방 찾을 수 있었고, 일부 파일명/폴더명이 특별히 다를 것도 없는데 다운로드 실패한 작품들도 PixEz의 작업 실패 목록으로 바로 확인 가능했다. 이조차 Pixiv-Shaft보다 가독성 좋고 확인 빨라.

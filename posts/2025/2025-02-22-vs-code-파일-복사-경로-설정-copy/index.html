VS Code 파일 복사 경로 설정

블로그를 옵시디언에 집어넣어버릴까 고민하는데 복사한 첨부파일을 자동으로 특정 폴더에 붙여넣어주는 기능이 부러웠다. VS Code로도 당연히 될 것 같은데 코파일럿한테 물어보고 한참을 확장만 뒤적이고 있었는데 마크다운 기본 확장의 기본 기능이었네?

https://stackoverflow.com/questions/75831497/how-can-i-paste-images-into-my-markdown-files-in-vs-code

Markdown › Copy Files: Destination
항목: **/*
값: images/

윈도우 탐색기에서 파일 선택하고 복사 후 작성중인 마크다운 문서에 붙여넣으면 현재 경로의 하위 images 폴더에 붙여넣고 첨부해준다.

기존 폴더분리하고 직접 파일넣고 경로넣는방식도 물론 잘됨 휴고 오케이이

vs code images/ ./images/ 둘중 뭘 줘도 vs code내에서 잘 보이고 그대로 옵시디언 던져줘도 마찬가지지 ../images/은 이미지 인식 못함 옵시디언에서는 인식함

https://lrk567301.github.io/posts/2025/2025-02-22-vs-code-%ED%8C%8C%EC%9D%BC-%EB%B3%B5%EC%82%AC-%EA%B2%BD%EB%A1%9C-%EC%84%A4%EC%A0%95-copy/images/1nonomiya-1893608214836519223-20250223-01.jpg 대신 이대로는 휴고가 안됨 휴고는 각 post 폴더 아래 images 경로가 잡히니까

../images/ 옵션 주면 한칸 위 경로에 폴더 만들고 복사해넣음

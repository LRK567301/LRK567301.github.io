벗어날 수 없는 조합 Keep+OneNote+Wordpress

쿨타임 찰 때마다 더 나은 조합, 혹은 더 통합된 조합이 없을까 고민하고 도전해보곤 하지만 간단한 메모 킵, 자료정리 원노트, 블로그 워드프레스에 보조로 시놀로지의 노트 스테이션 조합을 벗어날 수가 없다. 벗어날 필요도 없고.

Joplin

가장 그럴싸한 대안. 에버노트 대체를 표방하는만큼 실제로 원노트→에버노트→조플린 마이그레이션을 해 봤을때 꽤 괜찮은 결과물이 나왔고, 웹 클리퍼, 윈도우/안드로이드 클라이언트, 클라우드를 통한 동기화까지 지원해서 가장 사용성이 높다. 한글 입력할 때 요상한 문자 뜨던 것도 해결됐네. 하지만 마크다운인건 좋은데 검색이 불편. 짧은 노트 검색은 괜찮은데 긴 노트를 검색하면 검색어 강조가 마크다운이 아닌 미리보기 창에서만 돼서 실제 편집점을 찾는게 어렵다. 문단은 괜찮은데 코드나 링크 나열이 되면 환장해. 그리고 Selfhost 클라우드나 WebDAV까지 지원하는건 좋은데 정작 문서명이나 첨부파일명이 자체 디비 기준이라 또 개판. 원노트나 에버노트라 생각하면 전혀 불편함이 아닌데 파일이 직접 내 눈에 보이니 신경쓰인다.

Notable

깔끔한 에디터, 깔끔한 파일관리 덕에 가장 마음에 드나 동기화와 모바일 미지원. 첨부파일을 비롯한 파일명이 원본 그대로인 덕에 외부 클라우드와 연동시키기도 보기 좋고 Noteless 같은 외부 모바일 앱도 있어서 Joplin만큼 올인원은 아니지만 어느 정도 보조가 된다. 폴더 구조 덕에 hugo 같은 정적 사이트 생성기와 엮어서 원노트+워드프레스의 꿈도 살짝 꾸게 해 줬고. 하지만 검색 왜이래? 조플린은 그나마 미리보기 창에라도 검색어 강조를 해줬는데 얜 검색어가 있는 문서명만 목록에 필터링해주고 있어 ㅋㅋㅋ 이게 검색이냐 ㅋㅋㅋㅋㅋㅋ

Standard Notes

윈도우 클라이언트가 조플린 깔끔해진 버전. 용량은 비슷하게 100MB 정도. 그런데 한글 입력이 치명적이다. 한글자씩 뭉쳐. 제목 쓰다 본문으로 넘어왔더니 제목 마지막 글자까지 본문에 따라왔네? 오픈소스고 도커 싱크서버도 있는데 익스텐션을 비롯해 유료 모델부터 눈에 띄는 것도 애매.

Trilium

모바일 페이지도 잘 지원하는 웹 노트. 도커로 브라우저에서 열어도 윈도우 클라이언트를 실행해도 하나의 서버 인스턴트. 첫 노트가 될 수도 있고 동기화될 수도 있고 재밌는 구조야. 노트로서도 강력하다. 잠깐 만져보니 가벼운 VS Code, 가벼운 노션 느낌이야. 언어별 구문 강조도 지원하면서 마인드맵 같은 노트 심화형 기능도 있어. 하지만 내부 파일 구조를 모르겠다. 마크다운/html로 페이지 개별 내보내기는 되지만 너무 독립적인 노트라 외부 접근성은 떨어져.

QOwnNotes

ownCloud/Nextcloud의 Notes 앱과 동기화되는 마크다운 노트. 웹에서는 넥스트클라우드로, 모바일에서는 넥스트클라우드 노트로, 윈도우에서는 QOwnNotes로 심화된 글쓰기가 가능하다. 접근성이 제일 높아. 어차피 노트는 텍스트 파일로 저장되니 넥스트클라우드도 sqlite로 쓰면 돼서 도커 구성도 부담이 없다. 그런데 아이고, 얘도 한글 입력이 한글자씩 뭉치네. 넥스트클라우드 노트는 안 그런데…. 넥스트클라우드 노트 자체도 아주 가벼운 노트라 분류가 간소하고 보기 살짝 불편해. 매번 열었다 닫았다. 파일 기반인데 히스토리를 지원하니 수정파일이 임시폴더에 엄청나게 쌓이는게 그대로 보여서 괜히 불편하고.

code-server (VS Code in the browser)

VS Code를 마크다운 에디터로 쓰는게 생각보다 훨씬 강력하길래 잠깐 외도. 마크다운 에디터 중에 이미지를 클립보드로 이렇게 쉽게 붙여넣을 수 있는 건 처음 봤다. Notable처럼 기껏해야 첨부파일 지원 정도지 한땀한땀 이미지 주소를 따서 넣어줘야 했는데. 하지만 그건 윈도우에서의 이야기고 code-server에선 불가. 사실 이미지를 복붙해봤자 내부 경로라 의미없지. 웹에서만 실행해도 꽤 강력한 에디터고 도커에 마운트한 폴더 구조를 그대로 접근 가능한 것도 참 좋긴 한데 노트 테이킹 앱은 아니다. 모바일은 어쩔거야. 그럴싸하게 뜨긴 하는데 모바일 웹에서는 좀……

Mark Text

파일첨부고 뭐고 아무것도 없는 정말 깔끔하고 예쁜 단순 마크다운 에디터. 마크다운 에디터만 필요하면 얘를 쓰면 되겠다.

BookStack

설치는 안 했지만 흥미로워서 데모만 만져봤다. 서가와 서고로 정리하는 위키. 큰 위키 항목이 아니라 책이 한권씩 있고 거기에 내역이 나열된다. 재밌어. 예쁘고.

역시 가장 현실적인 대안은 조플린이다. 하지만 대안은 대안에 불과해. 노트로 (그나마) 뛰어난 조플린은 파일관리가 독자적이라 외부연동이 불편하고, 외부연동하기 딱 좋은 노더블은 검색이 병신이네. 물론 정적 사이트 생성기를 가져와서 어찌어찌 연동하는 구조를 갖춘다 해도 그게 이미 불편하기 짝이 없다. 깃헙 페이지 덕에 워드프레스가 느리다고 많이 까이는데, git push가 일상이라 왜 CMS인지는 고려 안 하나봐. 관리, 검색, 첨부파일, RSS, 접근성, 에디터, 태그, 이런 관리 편의성 차이가 크고 작고 이전에 비교할 수 조차 없는걸. 물론 단순히 결과물만 봤을 때의 속도나 서비스에 종속되지 않은 파일 구조로 원본 데이터를 그대로 다루는 매력, 자유로움은 인정하기에 잊을만하면 이 고생을 하고 있는 거지만 ㅋㅋㅋㅋㅋㅋㅋ

물론 그 빈틈을 메워줄 수 있는 것도 존재한다. 헤드리스 CMS. 글 관리와 글 보여주기의 분리. 정적 사이트 생성기와 연동하는 모바일 워크플로우(이게 검색어였어!)도 템플릿 파일 하나 만들어두고 폰용 마크다운 에디터로 클라우드에 바로 써넣으면 hugo 서버 같은게 자동으로 인식해서 페이지 만들어줄 수도 있겠고. 하지만 이름을 몇번 들어본 Netlify CMS의 검색 수준이 기껏해야 제목 필터링인걸 보고 차게 식었다. 이렇게까지 써야 해? 도커 덕에 편해졌다지만 그래도 구축과 관리에 노력이 너무 많이 든다.

그래도 자꾸 미련이 생기는건 10년간 다른 형식으로 써온 원노트와 블로그 글이 두 플랫폼의 범용성과 마크다운 덕분에 어떻게 어떻게 통합이 되려면 될 수도 있을 것 같아서야. 원노트는 의외로 쉽다. 이쪽엔 첨부파일도 사진도 없고 기껏해야 테이블이니 에버노트를 징검다리 삼아 어지간한 서비스에는 넘어갈 수 있어. 시놀로지 노트 스테이션으로 갈 수도 있을 것 같아서 무섭지.

워드프레스도 그렇다. 이쪽은 더 쉬워. 이미지 외에는 따로 첨부한 파일이 없고, 그 이미지는 마크다운 프리뷰로 볼 수 있다. 분류와 태그만 살릴 수 있으면 돼. 아마 그게 문제일 것 같지만.

아으으 정말……….진짜 이상적인 툴이 하나 있었으면 좋겠어. 일반 텍스트 글과 이미지를 삽입한 출판용(?) 글을 한 곳에서 쭉쭉 쓰다가 외부로 공유하고 싶을 때만 링크 따서 트윗. 자기만족과 기록의 글쓰기 중에 공유만 한번씩 하면 되거든. 원노트의 공유버튼이 아무런 의미가 없는게 참 아쉽다. 이거 당장 에버노트나 노션에 진입하면 할 수 있는 거거든. 그런데 이제와서 그런 무거운 서비스에 발을 들인다? 원노트와 킵을 쓰면서 그게 제일 좋다는 건 알고 있지만 그건 못하겠어. 할 거면 클라우드 오피스를 쓰는게 나아 보일만큼.

그래도 설치형인 워드프레스를 제외하고 원노트나 포켓을 대체할만한 서비스가 보여서 다행이다. 물론 그걸 실제로 고려할 시점은 믿고 쓰는 대형 서비스가 망해 없어질 시점이겠지. 내가 죽을 때쯤이 아닐까? 그때까진 편한 걸 쓰자. 도구가 아무리 잘나봐야 도구, 도구는 쓰기 편한게 최고야. 현재로써는 원노트를 탈출할 수 있게 해 주는 쿠션, 에버노트의 존망만 신경쓰면 되겠어.

그나저나 회사에서 깃헙은 갑자기 왜 막혔대? 에버노트 같은데 막힌건 그렇다치고 깃헙이 왜….이직 사이트 취급인가? 도커 허브로는 부족한 내 즐거움을 돌려줘….

code-server로 블로그 갈아타려다 워드프레스 테마 변경

또 쿨타임 찼다. 워드프레스 유지하기가 갑자기 귀찮네. 게시판 vps에 옮겨버리고 파일로 글 관리하면 백업도 접근성도 훨씬 좋아지지 않을까? 도커로 마리아DB는 그렇다 치고 php 웹서버 환경을 구성하기가 좀 귀찮아보이긴 하지만.

code-server는 역시 linuxserver 이미지. 본가 이미지는 몇번 해봐도 실행이 안 돼. 볼륨 제대로 잡은 것 같은데.

wordpress-export-to-markdown는 꽤 많은 업데이트가 있었다. 태그는 없지만 카테고리도 들어가고 한글 url 디코드 처리도 됐어. 시간 포기하면 이걸로 충분히 깔끔한 결과물. 워드프레스 xml 내보내기할때 글만 내보내는 거 잊지 말자. 메모리도 잡아줬는데 왜 파서 오류가 나나 했어.

코드블럭이 날아가는게 아쉬워서 wordpress-to-hugo-exporter도 테스트. 워드프레스를 도커로 옮긴걸 잊고 헤맸다. 워드프레스 플러그인에 넣고 워드프레스 컨테이너 안의 php로 실행해야지. 헤더&문단 구성과 코드 블럭은 여전히 깔끔하지만 이미지 처리가 개판. 그냥 마크다운 문법으로 바꿔주지 왜 img에다 크기까지 지정해둬서 vs code 프리뷰에서 비율이 깨져 보이게 만들어. 코드블럭 포기하는게 낫다.

며칠 code-server를 써보니 정말 괜찮다. 접근성 좋고 편집 빠른게 너무 편해. 오프라인 에디터를 따로 설치할 필요가 없다. vs code가 통째로 올라가있는데 이 이상 무엇이 필요? notable은 년도별로 나눈 하위 폴더를 인식하지 못해 무쓸모. 이미지 업로드가 좀 귀찮겠지만 글쓰기는 탐색창에서 복붙 F2 단축키 다 쓰며 파일을 오프라인처럼 다룰 수 있으니 끝내줘.

문제는 모바일. 데스크탑…..은 이제 없구나. 노트북-탭까지는 딱 좋아. 근데 모바일…….쓸 수는 있는데, 쓸 수야 있는데 글 하나 보려고 데스크탑 UI를 어거지로 패널을 닫았다 열었다 하며 쓰는게 고통. 시놀 앱으로 접근해서 텍스트로 열어보는게 나아. 이건 좀 더 찾아볼 필요가 있다. DS File에선 파일명 검색만 되는 것도 그렇고.

워드프레스를 치우면 그 데이터를 라이믹스 게시판에 옮길까도 했는데 어찌 될듯 말듯 귀찮군……앞으로 써 가는 글만? 반쯤 버리는 걸로 생각하면 괜찮지. 하려면 라이믹스를 워드프레스로 옮겨야 하는 시대니까.

쿨타임을 비우면서 문득 드는 ‘워드프레스 테마를 바꿀까?’ vs code에서 글을 잠깐 써보니 썸네일까지 열심히 구성해둘 필요가 없다. 좀 더 텍스트 위주로 깔끔하고 에디터스러운 모양으로. 워드프레스만한 에디터/CMS가 또 없어.

McLuhan. 이번에도 by Anders Norén. 이 제작자의 Iwata를 최소 2018년부터, 혹은 워드프레스를 시작했던 2016년부터 쓴 것 같은데 다시 돌아봐도 이 사람만큼 마음에 드는 테마가 없다. 한글 글꼴 문제조차 없어. singular.php에서 post_thumbnail만 지워주면 끝. 약간 애매한 컬러값이 남아있는 걸 보니 한번 외도했던 테마가 맞는데 그때 느낌이랑 지금이랑 다르다. 마음에 쏙 들게 깔끔해.

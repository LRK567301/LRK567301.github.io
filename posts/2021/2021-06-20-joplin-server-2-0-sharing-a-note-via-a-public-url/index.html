Joplin Server 2.0: Sharing a note via a public URL

조플린 서버 2.0이 노션처럼 외부 URL 공유를 지원한다고? 동기화도 훨씬 빠르다고? PostgreSQL을 쓰면 더 빠르다고? 웹은 없지만 모바일 앱도 있고 공유도 되면 워드프레스 쓸 필요 없지 않아?

https://registry.hub.docker.com/r/florider89/joplin-server/

공식 이미지는 설정을 환경변수가 아니라 env 파일로 가져오길래 간편한 비공식 이미지로. 2기가나 되는 건 똑같군. PostgreSQL 안 쓸거면 APP_BASE_URL이랑 APP_PORT만 잡아주면 된다. 기본 계정은 admin@localhost / admin.

https://github.com/stanlemon/wordpress-export-to-markdown

이번에도 워드프레스 데이터는 node 도커 이미지를 통해 export. settings.js에서 시간도 가져올 수 있는 걸 이제서야 알았다. 한글 slug와 태그까지 이제 너무 잘 가져와서 더 바랄 건 없다고 생각했는데.

docker exec -it 97f06266b854 /bin/bash
npm install
node --max-old-space-size=3072 index.js

잠깐 만져보니 그럴싸하다. 동기화가 좀 애매한 느낌(윈도우에서 노트를 동기화해 올린 후 안드로이드 설치, 기본 노트를 지우고 동기화를 누르니 서버를 싹 비워버린다?)도 있었는데 윈도우에서 다시 노트를 추가했더니 안드로이드도 따라왔고, 무엇보다 공개 URL 공유. 윈도우 클라이언트에서 공유를 누르니까 서버랑 동기화 한번 하고 주소를 복사해줘. 원노트를 쓰면서 노션, 에버노트가 부러웠던게 바로 이건데!

하지만 잘 동기화되는 건 워드프레스의 2021년도 글 87개만, 그것도 첨부 이미지를 제외한 상태에서만이었다. 10년치 글, 첨부 이미지까지 포함해 파일 13320 개 폴더 189개가 되니 윈도우에서 올리는 동기화는 시간이 걸릴 뿐 해 내는데 안드로이드에서 받아오질 못해.

물론 워드프레스에서도 모든 글을 공개하고 있는 게 아니니 공개분 몇년치만 잘라내면 조플린도 잘 동작하겠지만 검색과 공유를 포함한 노트 툴로서 워드프레스를 대체하고 싶은건데 의미가 없어지지. 이러면 URL 공유 말곤 webdav 동기화랑 다를 게 없잖아….

글 제목을 바꿨는데 md 파일명은 바꾸기 전인 게 있어서 유심히 보니 워드프레스 리비전 탓. 그래서 새삼 워드프레스를 둘러보니 내가 참 무심하게 썼구나 싶다. 블럭 에디터의 Reusable Blocks에도 이상한 블럭이 하나 있고, Stop Generating Unnecessary Thumbnails 플러그인을 씀에도 불구하고 테마에서 따라온 쓸데없는 썸네일이 또 있어.

코드블럭도 그래. 왜 wordpress-export-to-markdown의 결과물에 블럭이 있다 없다 하나 했더니 변환 안되는건 ‘사전 서식화(Preformatted)’, 변환되는건 ‘코드(Code)’. 하 씨 뭔데 이 어정쩡한 블럭…보여주는건 비슷해서 헷갈렸잖아. 워드프레스 임베드 너무 늘었어. 링크 좀 그만 변환해 작은 글쓰기 공간에 파라그래프 임베드 필요 없다고…..유튜브야 편하지만.

하지만 동기화 서버가 있어도 디비가 있어도 그래도 파일 단위로 한땀한땀 동기화하는 듯한 조플린을 보니 웹은 웹이고 앱은 앱임을 재확인. 상용 노트 서비스의 동기화는 과연 클라스가 다르고.

디비 없이 md 파일로 쉽게 관리하고 웹에서만 봐도 될 글을 로컬에 전부 동기화할 필요가 없다는 욕구만 따지면 역시 웹, Strapi+Gatsby 같은 헤드리스 CMS 조합이 딱일 것 같지만 세팅은 그렇다 치고 검색………Gatsby는 그나마 검색 튜토리얼이 있지만 정적 페이지의 자체 검색 수준이라는게 사실 ㅎㅎㅎㅎ 디비를 괜히 쓰는 게 아니야.

그리고 이미지 삽입도 그래. 현재 워드프레스처럼 년/월 폴더로 깔끔하게 알아서 담아줄 수 있는가? 그럴 리가 없을 것 같지? 역시 워드프레스가 선녀다. 괜히 많이 쓰는 게 아니고. 깃헙 링크 붙는 건 개발자의 영역. 일반인이 건들 게 아냐.

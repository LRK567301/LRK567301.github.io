[윈도우] 꿀뷰의 유료화에 놀라 이것저것 테스트

꿀뷰의 유료화에 놀라 다른 건 없을까 테스트.

NanaZip

7zip의 윈도우11 탐색기 메뉴 지원 개선판. 글꼴 표현도 깔끔하고 본체 UI도 훨씬 예쁘다. 하지만 반디집 미만. 탐색기 메뉴 내용물이 7zip 그대로라 반디집의 ‘여기에 풀고 삭제하기’란 개꿀 버튼은 못이겨.

XnViewMP

설정-일반-파일로 시작시 모드-뷰어를 선택하면 몇장 없는 폴더도 만장 넘는 폴더도 비슷하게 빨리 열어준다. 북스캔할 일이 없어지니 뷰어 열어둔 상태로 뒤쪽 파일 갱신 추가 신경쓸 것도 없고. 이미지 확대/축소시의 품질 아쉬움도 설정-보기-고배율 품질에서 변경 가능. 브라우저-뷰어 탭 여러개 연동은 여러 이미지 비교할 때 꿀뷰 같은 깡뷰어나 속터지는 윈도우 탐색기와 비교할 수 없이 좋고.

하지만 반디집-꿀뷰 연동은 못이겨. XnViewMP 자체가 압축파일 지원이라도 하면 모를까 꿀뷰 조합이 너무 미쳤어. 압축파일 열고 엔터 한번 누르면 꿀뷰로 압축파일을 그대로 이어주는게 쩔어. 다른 애들은 해당 사진파일 딱 하나만 임시로 압축 풀고 열어주면 윈도우 기본 연계 다 한거거든. 메뉴 한글 글꼴도 여전히 아쉽고. 파일 탐색에서는 현재 얘만한 애가 없지만 뷰어에서는 간편한 조작과 윈도우 창 확대되는 애니메이션조차 없는 속도와 성능 원탑의 꿀뷰.

nomacs

https://www.reddit.com/r/StableDiffusion/comments/yypynf/a_list_of_windows_image_viewers_and_managers/

AI짤 스레드를 둘러보다 이름을 또 봐서 테스트. 근데 이게 이미지 갤러리/매니저에 있는 게 맞냐….썸네일 뷰는 진짜 딱 폴더 내 파일들 썸네일만 보여주는 뷰. 탐색기처럼 채워서 보여주는 것도 아니고 갤러리처럼 크게 보여주는 것도 아니고 화면 중앙에 손바닥만한 창에 몇개 보여주는게 다야. 메뉴는 다양하지만 불편 그 자체.

JPEGView

꿀뷰는 신입니다 닥치고 그냥 꿀뷰 써라. 빠르고 깔끔하긴 한데 순정 mpv처럼 설정을 죄다 커맨드라인에 때려박아서 나온 깔끔함. 조작도 구리고 한글 글꼴도 반쯤 엉망이고 파일 적을때나 빠르지 만장 넘는 폴더 열면 꿀뷰처럼 멎는……..꿀뷰보다 한참 느린데? 오히려 꿀뷰가 예전보다 훨 빠르게 느껴져서 놀라운데? 절대 오픈소스만 쓰겠다는 신념이라도 있어야 가능.

QuickLook

이거 물건인데? 윈도우 탐색기에서 사진 클릭된 상태로 스페이스바만 누르면 파일이 적건 많건 초고속 미리보기 팝업. 팝업된 창은 맨 위에 있는 상태로 키보드 방향키로 뒤에 있는 탐색기창 파일 옮겨가는대로 선택되는 이미지를 계속 보여준다. 어케한거야 정신 나갈 것 같애. 주력….은 아니고 본격 뷰어와 탐색기 메뉴 팝업 중간 쿠션쯤 되는데 그 쿠션이 너무 강력해.

PhotoSift

https://www.rlvision.com/photosift/about.php

sdcompare와 비슷한 과의 툴. ABCDE 각 버튼에 폴더 지정해놓고 얘로 사진 휙휙 넘겨보면서 버튼 눌러서 분류할 수 있다. AI짤 덕분에 재밌는 거 많이 알게 돼.

foobar2000(안드로이드) + 시놀로지 미디어 서버

Navidrome는 좋지만 태그 수정 등 파일 관리할 일이 조금이라도 생기면 웹 UI 너머인게 불편하고 항상 %를 붙여야 하는 검색과 더불어 모바일에서 재생 큐 편집이 안되는 것도 괴로웠다. 울트라소닉처럼 처음부터 잘 고르는 편법 뿐인데 것도 실수하면 다 비우고 처음부터 시작해야 해.

어차피 외부/모바일에서도 푸바의 플레이리스트를 공유하고 싶어서였으니 시놀이나 푸바의 미디어서버 기능을 쓴다면…….은 꿈. 최신 푸바 2.0에서는 upnp 서버 접속할 방법도 없더라. 플러그인이 너무 옛날거야. 대신 안드로이드 푸바에서 시놀 미디어서버에 접근해보니 어휴…..되긴 된다만 앨범아트 품질은 240p쯤 되어보이는 초저퀄이고 플레이리스트 폴더가 있긴 한데 관리할 방법도 없으며 검색도 없다. 안 쓰이는 게 이유가 있군.

foobar2000(안드로이드) + foobar2000 1.6 + foo_upnp

구버전 푸바에서는 서버도 클라이언트도 가능하니까 일단 서버. 시놀 미디어 서버보다 훨씬 낫다. 앨범아트도 고해상도로 나오고 푸바의 플레이리스트들이 그대로 뜨는 게 커. 근데 그래봤자 upnp. 클라이언트에서 플레이리스트 만들어봤자 서버(푸바)의 플레이리스트와 격리된 자체 리스트일 뿐이고 검색은 여전히 못한다.

mpv.net

여전히 빠르고 강력한데 마우스 커서만 갖다대도 뜨는 너무 민감한 seek 바가 전체화면 상태에서 간혹 혼자 뜨기까지 해서 집중을 방해하고 재생 중에 윈도우 오디오 전환시 한참 묵음이 유지된다. 스피커-블투 전환할때 귀찮아.

MPC-HC

https://github.com/clsid2/mpc-hc

mpv보다 살짝 늦은 정도로 나스의 200MB도 2GB도 20GB 영상도 빠르게 잘 연다. 요즘 KMP는 괜찮은데 약간 전 KMP는 잘 열다가 20GB대에서 버벅였거든. UI 창이 너무 클래식인걸 제외하면 다크테마에 한국어 메뉴 글꼴도 깔끔해서 충분히 현대적이야. 단축키 설정 다 돼, 볼륨 노멀라이징 체크 한번만 하면 되니 간편해, seek 바 프리뷰까지 되니 mpv보다 낫나? 유튜브 못 여는 줄 알았는데 yt-dlp 넣어주니까 콜링 yt-dl 잠깐 뜨고 고급-YDLMaxHeight에서 지정된 해상도로 오디오까지 잘 나온다. 스크린샷 파일명은 건드릴 것도 없이 깔끔한데 저장 경로 지정을 못하는게 아쉬울 뿐.

MPC-BE

https://github.com/Aleksoid1978/MPC-BE

다크테마를 따로 켜는 HC와 달리 이름에 걸맞게 기본적으로 다크한 스킨. HC에서 ‘고급’ 페이지 하나 만들어서 걍 나열해버린 것들이 재생-온라인 미디어 서비스-유튜브로 세분화, 마우스도 단축키 설정 페이지에 통합되어있는 HC와 달리 별도 페이지인 등 진화판. HC 전체화면 OSD에서 시간 표기를 못 끄는게 아쉬웠는데 BE는 그 설정조차 세밀하게 돼. 전체화면에서 마우스 위쪽에 갖다대면 작은 버튼들까지 꼼꼼하고. 유튜브 재생도 내장된 웹 페이지 파서 사용과 yt-dlp 별도 사용 가능. mpv+yt-dlp는 해상도만 4k 최대로 뽑고 오디오는 안 나오는 영상을 HC+yt-dlp도 BE도 오디오까지 잘 재생해준다. 근본 계보는 HC가 맞는데 괜히 블랙에디션이라고 따로 이름 빼둔 게 아니구나. 이왕이면 근본을 쓰려고 했는데 세세한 편의성 차이가 커. 폴더 내 다음 파일이 자동 재생이 아니라 바꿔주는 등 커스터마이징은 필요.

KMPx64

여전히 유튜브 재생은 못하고 설정 메뉴도 지금보니 MPC 빼박이다. 캡처 경로 저장이랑 스킨이 쪼끔 현대적인 거 말고는 MPC 하위호환 껍데기. 메인에 기본 표시되는 광고와 도네 추천 버튼을 끌 수 있는 건 다행.

잠정 결론은 엉뚱하게 mpv.net에서 MPC-BE로 교체? 이미지 뷰어도 기본 XnViewMP 고정해두기 꽤 괜찮더라. 꿀뷰에 있는 기능 정도는 얘한테 다 있고 이미지 한장 연 다음에 폴더 탐색으로 넘어가는 과정이 속도 포함해 너무 좋거든. 압축된 이미지 보기에서 반디집+꿀뷰 조합은 신이지만 기본 뷰어+탐색 XnViewMP, 최신 정렬은 윈도우11 사진이 받쳐주면 더 찾을 것도 없지 않나. 압축파일까지 지원하는 ACDSee 구매도 하나의 수단이지만 걔는 싱크 어쩌구에 이것저것 무겁게 깔리는데다 반디소프트 생태계가 너무 깔끔하면서 뛰어나.

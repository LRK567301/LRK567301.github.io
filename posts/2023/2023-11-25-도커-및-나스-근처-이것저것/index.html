수제 HTML 대시보드

못생겼어…..쿄우 깔아놔도 답이 없어 ㅋㅋ; HTML 기본 목록을 그대로 썼더니 뷰포트로 모바일에 폭 맞춰봤자 글자 너무 작아서 터치도 힘들고. 그리고 브라우저마다 처음 한번이면 충분한 헤임달 패스워드 인증과 달리 HTTP 베이직 인증은 심심하면 다시 해야 했다. 이게 생각보다 많이 귀찮아.

Heimdall

디즈니플러스 덕에 마침 토르 시리즈를 클리어하고 있어서 정이 붙을 뻔 했으나 크롬 북마크에 NAS 폴더 하나 만들고 정리. 외부에서 자주 붙는 도메인은 기껏해야 한두개라 확실히 기억하고 있으니 따로 대문을 굴릴 필요가 없다.

Navidrome

와이어가드+VLC(SMB)라는 미니멀리즘의 광풍에 쓸려나갔다. 안드로이드 VLC가 목록 단위 영상 재생까지 좋으니 사용빈도상 필요가 없어.

STFPGo

무조건 강제되는 유저명_파일명.zip 압축 다운로드, 모바일에서 숨겨지는 파일명 뷰, 파일 이동은 한번에 못하고 무조건 경로 직접 타이핑해서(타겟까지 / 한번 더 입력하고) 복사 후 삭제, 공유기 와이어가드에서 smb 접속할 방법을 알고 나니 확 떨어진 sftp 활용도. 의외로 대단했지만 의외였을 뿐.

Filestash

https://github.com/mickael-kerjean/filestash/issues/222#issuecomment-607276633

온리오피스가 외부에 노출될 필요조차 없게 파일스태시 자체가 역방향 프록시 역할을 하는 완벽한 구성이라는데 난 아무리 해봐도 온리오피스 창은 뜨고 다운로드 페일이다. 밖에서 글쓰려는데 양식 파일을 복사할 방법이 없어서 분노하다 그냥 새 파일 만들고 내용만 복붙하면 된다는 사실을 뒤늦게 깨달았지.

덕분에 또 비교되는 사용성. 파일브라우저가 느려서 그렇지 Ctrl+C, V도 되고 파일 이동할때 경로 팝업도 뜨고 인터페이스가 진짜 좋아. 파일스태시는 진짜 최소한만 갖춘 성능충. 로드맵 보니까 안그래도 빠른데 레거시 걷어내면서 더 빨라지려고 하더라.

Misskey

https://qiita.com/nexryai/items/78e272d9bc36317d728a

https://hide.ac/articles/t6Gdoxdfh

https://blog.vcborn.com/2022/07/windows%E7%92%B0%E5%A2%83%E3%81%A8%E5%85%B1%E5%AD%98%E3%81%97wsl%E4%B8%8A%E3%81%ABmisskey%E3%82%A4%E3%83%B3%E3%82%B9%E3%82%BF%E3%83%B3%E3%82%B9/

도대체 미스키는 어떤 환경을 갖춰야 빌드가, 아니 이미지 실행이 되는거야? 시놀로지 안 돼, 오라클 VPS 안 돼, 도커 데스크탑 안 돼, WSL 안 돼, 우분투 VM 안 돼…….하이퍼V에 올린 우분투 VM에서는 돼야 하는거 아냐? 뭐 되는 게 없냐? 빌드하다 터지는 것 같아서 build essential도 설치했잖아 ㅋㅋ 그래도 안 돼서 이미지로 시도하니 우분투 VM에서만 간신히 돈다. 미친다 진짜 ㅋㅋ

    volumes:
      - ./files:/misskey/files
      - ./misskey.yaml:/misskey/.config/default.yml:ro

이렇게 간단한데도 안돼서 한참을 삑삑댔는데 쿨타임 두고 다시 잘 보니 ./misskey.yaml가 문제였다. 볼륨 마운트로 파일 인식을 못해. misskeyconf 등으로 바꿔버리면 WSL에서도 잘 된다.

https://github.com/misskey-dev/misskey/blob/develop/docker-compose.yml.example

build: .
image: misskey/misskey:latest

빌드를 이미지로 변경은 필수.

간신히 뚫어보니 역시 마스토돈보단 미스키다. 이메일 인증부터 없어도 돼. 근데 데탑 성능으로 굴리니 정말 빠릿하고 좋은데 매 분기 신작처럼 길고 큰 이미지는 원본을 제대로 보여주지 못하고 재부팅시 web 컨테이너가 한번에 못 올라와서 수동 재시작해줘야 하는 경우가 있어.

안드로이드 마크다운 에디터 비교

Markor

프리뷰 기본 뷰, 외부 url 이미지 읽기, 폴더 선택 열기, 원드라이브 등 외부 프로그램에서 연결까지 잘 돼서 안드로이드 최고존엄이 될 줄 알았더니 파일을 열 때마다 안드로이드 멀티태스킹 창이 늘어난다? 원핸드 오퍼레이션으로 빠르게 창을 오갈 때 환장하게 거슬려 워크플로우를 박살내 ㅋㅋㅋ 뭐 이런 끔찍한…..아 아니구나 기본 설정에 ‘다중 창’ 옵션이 켜져있구나?

그 외에도 설정이 쪼끔 필요하다. 보기 모드에서 편집 대신 읽기 모드 우선, 마크다운 형식에서 Newline = new paragraph 체크. 그래도 본문 검색으로 찾아들어가면 읽기가 아니라 편집 모드긴 하지만 제텔이나 옵시디언과 달리 완벽하게 순수 파일 기반이다. 하위폴더를 만들어가며 라이브러리화하는게 아니라 매번 직접 파일을 검색해. 덕분에 작업 폴더가 깔끔하게 순수하다.

SimpleMarkdown

폴더 열기 같은 거 없이 정말 본분에만 충실한 에디터. 주력이 될 뻔도 했는데 외부 url 이미지는 화면에 맞게 확대축소가 안 되고 폴더를 못 읽으니 ../folder/pic.jpg 같은 내부 경로 이미지도 아예 보여주질 못해서 탈락.

neutriNote

다른 앱으로 열기를 통해 외부 url 이미지까지 보여주는 뷰어로서는 나름 준수한데 삼성 파일 앱에서 md를 한번 열면 빠져나갈 수가 없어서 창(실제로는 파일 앱) 자체를 닫아야 하고 폴더 지원이 안 돼서 내부 경로 이미지도 못 읽는다.

Zettel Notes

다른 앱으로 열기, 외부 url, 내부 경로까지 퍼펙트한데 노트 폴더를 라이브러리화해서 파일본문 검색까지 순식간이고 markor와 달리 본문 검색으로 파일에 접근해도 읽기 모드를 유지한다. 첨부된 내부경로 이미지 누르니까 다른 앱으로 열어주기까지 하는데? 디자인도 공대스러운 markor와 달리 최신노트스럽게 매끈하고 진짜 최고존엄 후보. 근데 한글 사이 간격이 이상하게 넓다. 윈도우에서 한글 제대로 표현 못하는 글꼴로 억지로 보여주는 듯한 느낌.

Obsidian

윈도우 UI롤 모바일에도 그대로 가져온건 대단한데 그게 모바일에서 편한 건 아니다. 검색 어딨는지 한참 찾았잖아. 파일 탭 밖으로 한번 나가야 서치 태그 북마크가 있어. 볼트 열기만 지원해서 다른 앱으로 열기에 뜨지도 않고 데탑 폴더 그대로 단방향 동기화하니까 .obsidian 폴더가 충돌나서 모바일에 열어뒀던 탭이 닫히기도 하고 애매한 면도 있지만 폴더를 라이브러리화해서 본문까지 검색 빠른건 좋다. 검색에서 읽기모드 기본값 유지도 되고 유료지원도 있으니 디자인도 제일 좋고 검색도 잘 보니 하단메뉴 퀵 스위처에서 빠르게 가능해.

Logseq

개념이 어렵다. 일반 노트가 아니야. 저널, 블럭? 단위 메모 및 정리. 옵시디언 그래프 뷰도 안 쓰는데 훨씬 더 선을 넘었어.

옵시디언도 치우고 윈도우 VS Code 워크스페이스, 안드로이드 Zettel Notes 조합이 끌렸다. 하지만 옵시디언의 윈도우/안드로이드 동일한 위지윅 읽기&편집 사용자 경험의 편의성 및 깔끔함, Zettel의 단어 간격 때문에 애매하게 구린 가독성 때문에 보류. 그렇다고 옵시디언만 쓰려니 다른 앱으로 열기 지원을 안해서 결국 markor로 돌아가게 돼. 일단은 폴더싱크 동기화에서 .obsidian, .zettel-notes 폴더부터 필터링하자. vs code+위지윅 확장 & markor 같은 하위 폴더 만들 필요 없는 구성도 생각해봐야겠네.

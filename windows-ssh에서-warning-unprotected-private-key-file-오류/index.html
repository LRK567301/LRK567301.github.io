[Windows] SSH에서 WARNING: UNPROTECTED PRIVATE KEY FILE! 오류

282811-120-23T1211:128:400+09:00

그래 맞아, putty→윈도우 터미널→putty 한바퀴를 돌았던 이유 중 하나가 이거였어. 나스에 있는 key 파일을 못 읽는거. 몇번 시도하다 서버마다 배치파일 따로 실행하기도 귀찮아서 다시 putty를 썼던건데, putty 초기 설정하기는 세션별 글꼴까지 바꿔줘야 하니까 더 귀찮아서 남겨둔 배치파일을 열어보니 이제 기억이 난다. 윈도우 터미널로 띄우겠다고 wt cmd /c ssh 붙여서 실행했었는데 이젠 그것도 필요없군. 윈도우 너무 좋아졌어. 그러니 있는 거 쓰자.

증상은 윈도우가 설치되지 않은 디스크: 나스의 공유폴더나 로컬이라도 다른 디스크에 있는 key 파일을 지정하면 bad permissions, Permission denied. 공유폴더만 확인했을 땐 네트워크 폴더니까 그럴 수도 있나 하고 넘어가버렸는데 로컬의 다른 디스크에서도 그러는 걸 보고 그게 아니구나 싶었다.

https://superuser.com/questions/1296024/windows-ssh-permissions-for-private-key-are-too-open/1296046

로그는 정직했어. 정말 퍼미션 문제였네. key 파일의 상속을 제거해서 모든 권한자를 없앤 후 ‘나’만 추가하니까 언제 그랬냐는듯 잘 붙는다.

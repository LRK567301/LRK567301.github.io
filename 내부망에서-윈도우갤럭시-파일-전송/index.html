내부망에서 윈도우→갤럭시 파일 전송

17175-110-17T115:118:60+09:00

오래 잘 써 온 MiXplorer가 이번 업데이트에서 FTP 익명 로그인을 제거했는데, 새로 기본값으로 준 Admin으로 로그인이 되지 않길래 잠시 방황.

Solid Explorer 퀵패널 FTP 바로가기도 생겼고 모바일 UI 완성도는 날것인 믹스랑 비교도 안되게 좋은데 네트워크 쪽은 불안정했던 기억이 사라지질 않아. 업데이트도 8월 이후 없네.

사용자 휴대폰 도우미(Windows) 이것저것 기능 많은 건 좋은데 파일만 전송하긴 매우 불편. 폰 퀵패널에서 Windows와 연결 켜고 데탑에서도 프로그램 켜고 양쪽이 잘 붙었으면 데탑에서 폰 미러링 화면을 열어서 거기다 던져야 해. 그리고 폰에서 다시 필요한 폴더로 파일 옮기기.

Samsung Flow 파일/텍스트 전송 특화라 윈도우 연결 앱보다는 훨씬 편하다. 폰/데탑 양쪽 앱 켜고 던지기만 하면 되니까. 하지만 폰에서 다시 파일을 필요한 폴더로 옮겨야 하는건 마찬가지. 갤럭시 전용인데다 사용자 휴대폰 도우미처럼 클라이언트마다 설치가 필요한 것도 크고.

Snapdrop 에어드랍에서 삘을 받았다는 웹브라우저를 통한 로컬 공유 툴. 로그인이고 뭐고 없이 전송하고픈 기기 간에 페이지를 열고 던지면 된다. 외부 페이지인게 찜찜하면 도커 컨테이너(왠지 안 됐지만)도 있고. 그리고 웹 혹은 모바일 앱에서 저장, 다시 필요한 폴더로 파일 옮기기. 나쁘지 않아. 아니 좋아. 근데 나스에 로컬 전용 컨테이너를 올릴 거면 그냥 나스에 복사하고 옮기는게……?

나스의 자주 접근하는 폴더 루트를 임시폴더로 쓰곤 했는데 폴더를 하나 더 만들었다. 거기다 푸바에서 곡을 복사하고 폰에서 ‘내 파일’로 접근해서 ‘이동’. 네트워크 폴더는 즐겨찾기가 안 되는게 아쉽지만 로컬 폴더들은 즐겨찾기가 되니까 못 할 일은 아냐.

아예 나스에 폴더를 만들고 보니 여기서 FolderSync를 심화 활용(폰/탭 또는 음악/사진 하위 폴더를 만들어서 데탑에서 각 기기에 주고 싶은 파일 복사해두면 폰/탭 폴더싱크들이 동기화)하는 것도 괜찮아보이긴 하는데 이게 뭐라고 그렇게까지 하나 싶어서 참았어. 중복 파일 처리도 필요할테고.

관성이란게, 습관이란게 참 무서워. 폰에 FTP 서버 열고 데탑에서 파일질라로 필요한 폴더에 바로 쏴주는게 편하긴 한데, 그렇게는 한달에 한두번 음악 전송밖에 안 하면서 폰에서 ‘내 파일’로 나스 사진 가져오는건 매일같이 하고 있었으니 이게 무슨 비효율 ㅋㅋ 뭐하러 나눠서 했어 귀찮게…..한번씩 리셋이 필요해.

[워드프레스] 테마가 지정하는 에디터 폭 수정

2021-03-21

잘 쓰고 있는 McLuhan 테마의 블록 에디터 가로 폭이 갑자기 늘어났다. 테마 버전은 그대로(2.0.5/2020년 5월 26일)니 의심가는 건 워드프레스 업데이트. watchtower 컨테이너가 알아서 업데이트해주는 덕을 역으로 봤나 보다. 과연 잘 돌아가는 서비스는 손대는 게 아니구나.

구글링해보니 테마의 functions.php와 style-editor.css에 커스텀 css 를 추가하는 방법이 있는데 너무 과해. 하지만 혹시나 해서 테마 편집하기의 css 메뉴에 추가하는 것만으론 안 되더라.

워드프레스가 업데이트되었어도 여전히 잘 작동하는 Iwata(2.0.0/2020년 5월 12일)와 유심히 비교했다. wp-block 유무인가?

/assets/css/mcluhan-block-editor-styles.css

Structure의 .editor-styles-wrapper 항목 3개가 max-width: 590px;길래

.wp-block {
 max-width: 590px;
}

를 추가했다. 폭이 돌아왔다. 그런데 그래도 묘하게 넓다. 본문 폭이랑은 달라. 테마 스타일시트의 Structure를 보니 section-inner가 560px이다. 530까지 줄이니 이제 좀 맞아.

이유는 모르겠다. 워드프레스 5.7이 되면서 테마 문법이 깐깐해졌던지 버그가 생겼던지 둘 중 하나겠지. 테마의 본문 폭이 에디터랑 1:1 동기화된다는게 매우 합리적이면서도 귀찮아. 바로 이럴 때.
